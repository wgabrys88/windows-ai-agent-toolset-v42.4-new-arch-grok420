Franz - Logical Architecture Analysis

Violations of the "Python is pure plumbing" rule -- places where Python makes decisions, substitutes data, or silently recovers instead of surfacing failures to the swarm

Issues

#1 JSON parse fallback silently swallows Grok's failure brain_agentic.py

Python substitutes {} when Grok produces malformed JSON. A failed or hallucinated Grok response silently becomes "no actions, no progress, no completion" and the loop continues as if Grok decided to do nothing. The swarm never knows this happened. The raw string should be fed back into the swarm as an error signal so agents can self-correct.

except (json.JSONDecodeError, ValueError):
    hub.log_event("Captain JSON parse failed", "warn")
    decision = {}  # Python invents a decision
#2 Python decides phase transition from action list length brain_agentic.py

Python is making a phase transition decision based on whether the actions list is empty. This is intelligence. The swarm should own phase state entirely. Grok must explicitly set the phase in its JSON output and Python must write it verbatim.

state["phase"] = "EXECUTE" if actions else "EVALUATE"  # Python deciding phase
#3 _dispatch_action provides default coordinate fallbacks brain_agentic.py

If Grok omits x/y from a click, Python silently clicks 500,500 (screen center). If Grok omits drag coords, Python silently drags 400,400 -> 600,600. These are invented actions the swarm never decided. A missing coordinate should be a hard failure surfaced back to the swarm, not silently corrected.

x = int(action.get("x", 500))   # invented coordinate
y = int(action.get("y", 500))   # invented coordinate
int(action.get("x1", 400)), int(action.get("y1", 400)),  # invented drag start
int(action.get("x2", 600)), int(action.get("y2", 600)),  # invented drag end
#4 press_key default fallback invents a key brain_agentic.py

If Grok omits the key field, Python presses enter. This is Python executing an action the swarm never chose.

hub.actions(hub.press_key(str(action.get("key", "enter"))))  # Python invents "enter"

Python is correcting a malformed Grok output (list instead of string). This masks a model compliance failure. The swarm should see the error and self-correct on the next cycle, not have Python silently fix it.

#6 observation contains no actual screen data brain_agentic.py

The variable is named observation and passed to all 5 agents as {observation} in their system prompts -- but it contains zero visual information. The actual annotated frame (frame_b64) is only sent to Grok. The 5 debate agents are reasoning about the screen without seeing it. For a 2B VLM this is especially critical: agents are hallucinating screen state from text alone. The frame should be included in agent messages or the observation string must contain real screen description.

observation = f"Phase={state['phase']} Progress={state['progress']}% Goal={state['goal']}"
# No image. No screen content. Agents reason blind.
#7 history is only updated on goal completion, never during failure brain_agentic.py

On every failed or partial cycle, history stays as "Starting." or the last completion string. Agents receive stale history with no record of what was tried, what failed, or how many cycles have passed. For a 2B model this is fatal -- it has no episodic memory to reason from and will repeat the same actions indefinitely.

if is_complete and next_goal:
    state["history"] = f"Completed: {state['goal']}. Now: {next_goal}"
# On every other cycle: history is never updated. Agents are blind to failures.
#8 max_tokens=512 for debate agents causes silent truncation brain_agentic.py

Each of the 5 agents gets 512 tokens max. For a 2B model that needs to reason about a screen, plan actions, and write a useful debate contribution, 512 tokens will produce truncated or incomplete responses. Grok then synthesizes truncated inputs. The captain gets 800 tokens which is also tight for producing valid JSON plus reasoning. Truncation causes silent partial output with no error signal to the swarm.

result = await hub.call_vlm_agent(messages, temperature=0.5, max_tokens=512, ...)
# 512 tokens is insufficient for screen reasoning + action planning on a 2B model

#11 VLM exception handler returns empty string, loop continues silently franz_hub.py

A VLM network error, timeout, or server error returns an empty string to the brain. In _call_agent this becomes the agent's debate contribution.
